REST API

Platforma Library udostępnia klientom funkcjonalności w ramach API w oparciu o architekturę REST. Wykorzystuje do tego celu protokół HTTP wraz z jego metodami: GET, POST, PUT, DELETE. Elementy zostały podzielone na zasoby np. /books, /users, itd. Dostęp do poszczególnych zasobów następuje poprzez wywołanie wybranej metody HTTP na adres: https://java-library-api.herokuapp.com, dodając ścieżkę do wybranego zasobu.

Metody HTTP

  • GET: Wykorzystywana do pobierania danych. Wszystkie metody GET mogą być wołane wielokrotnie, gdyż nie modyfikują żadnych zasobów serwisu.
  • POST: Wykorzystywana do utworzenia nowego zasobu (np. utworzenie nowego konta
  • DELETE: Wykorzystywana do usuwania zasobów.

Reprezentacja danych

Wszystkie wysyłane i odbierane dane z API są w formacie JSON. Podstawowym elementem tego formatu jest pole, którego nazwa jest oddzielona od wartości pola dwukropkiem. Pola rozdziela przecinek. Każdy element obiektu JSON, zawiera się w nawiasach klamrowych. Tablice w formacie JSON są zapisywane z wykorzystaniem nawiasów kwadratowych. Wiecej na temat obiektów jakie są zwracane można się dowiedzieć TUTAJ

Ograniczenie liczby zapytań

W usłudze Library REST API obowiązuje główny limit nakładany na id klienta - 1000 zapytań na minute.

Gdy limit zostanie przekroczony kolejne zapytanie zwróci błąd TOO_MANY_REQUESTS. Należy wtedy odczekać minute przed wysłaniem kolejnego zapytania

Logowanie
/login

Opis

Metoda loguje użytkownika do serwisu. Aby się zalogować należy w 'body' podać JSON'a w postaci { 'username' : 'nazwa', 'password' : 'haslo'}

Metoda zwraca w body wartość 1 (jeżeli wszytko poszło dobrze) oraz NAJWAŻNIEJSZE zwraca token, który jest aktywny przez kolejne 5 dni.

Token jest przesyłany w nagłówku (header) o tytule 'Authorization'. Należy go przychowywać w bezpiecznym miejscu. Jest on niezbedny do uzyskania dostępu do metod, które wymagają uwierzytelnienia.

Rejestracja
/register

Opis

Metoda rejestruje nowego użytkownika do serwisu. Aby się zarejestrować należy w 'body' podać JSON'a w postaci { "username": "nazwa" ,"password": "haslo", "email": "email@com" }

Metoda zwraca w body wartość 1 (jeżeli wszytko poszło dobrze).

Po udanej rejestracji należy się zalogowac, aby uzyskać dostęp do całej biblioteki
